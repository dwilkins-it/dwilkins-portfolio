Building a Repeatable Acquisition Playbook

DoubleVerify · 2022–2024 · 200+ employees · 5 countries

The Situation

DoubleVerify was acquiring companies at pace — OpenSlate (2022), Scibids (2023), RockerBox (2024). Each brought different tech stacks, identity providers, hardware ecosystems, and employees across multiple countries. No established IT integration playbook existed.

The Approach

After the first integration, I documented every step and built a repeatable framework:

  • Day-zero access provisioning and identity migration
  • Hardware lifecycle — procurement, imaging, shipping across regions
  • Google Workspace migration and Slack workspace consolidation
  • Vendor contract consolidation and SaaS license reconciliation
  • Communication framework for acquired employees

Outcomes

  • ✔ 200+ employees onboarded across 5 countries
  • ✔ Zero productivity loss during integrations
  • ✔ Repeatable playbook across 3 acquisitions
  • ✔ C-level recognition for execution quality
  • ✔ 99% Day 1 delivery SLA maintained

From 0% to 60% Ticket Automation

DoubleVerify · 2022–2024 · 900+ tickets/month

The Situation

The service desk was entirely manual. Every access request, provisioning task, and lifecycle event required human processing. Ticket volume was outpacing team capacity during growth from ~1,100 to 1,600+ employees.

The Approach

Treated this as an engineering problem, not a staffing problem:

  • Approval workflows: Automated approval chains tied to manager hierarchies and role-based provisioning in Okta
  • Lifecycle automation: End-to-end JML processes from onboarding bot to termination device locking
  • AI-powered routing: Built models for ticket categorization and intelligent routing
  • Self-service: 150+ service catalog items and hiring manager access request portals

Outcomes

  • ✔ 60% ticket automation rate
  • ✔ ~2,700 manual hours eliminated annually
  • ✔ $135K+ annual labor cost avoidance
  • ✔ Team capacity freed for strategic projects

$500K SaaS Consolidation Strategy

DoubleVerify · 2024–2025 · 6 platforms · 1,600+ users migrated

The Situation

Post-acquisition tool sprawl: redundant project management platforms, overlapping communication tools, inconsistent licensing. No centralized visibility into actual usage versus contracted licenses.

The Approach

  • License tracking tooling: Built custom Chrome extension for real-time reconciliation across Lucid, ChatGPT, and Figma
  • Zoom → Google Meet: 1,600+ user migration with cross-departmental change management
  • Monday.com → Jira: Project management consolidation partnered with engineering
  • Usage-based negotiation: Real data from tracking extension informed vendor contract renegotiations

Outcomes

  • ✔ ~$500K in annualized savings
  • ✔ Unified technology ecosystem
  • ✔ Custom tooling for ongoing license governance
  • ✔ Data-driven vendor negotiation leverage
Compliance & Security SOX Controls
Project Lockdown — Zero SOX Findings

DoubleVerify · 2024 · Google Workspace, AD, Okta

The Situation

As a public company, DoubleVerify faced SOX compliance audits with existing control deficiencies around privileged access. Admin rights were broadly distributed, approval trails inconsistent, no systematic access reviews.

The Approach

  • Architected zero-trust access model for remote support teams
  • Built service desk governance tooling for Okta, Google Workspace, and Active Directory
  • Implemented approval-based workflows with complete audit trails
  • Created recurring access review cadences with automated reporting

Outcomes

  • ✔ Zero SOX audit findings since June 2024
  • ✔ Eliminated all prior control deficiencies
  • ✔ Scalable governance framework
  • ✔ Reduced privileged access surface area

Global Employee Lifecycle Automation

DoubleVerify · 2023–2025 · 7 lifecycle events

The Situation

Employee lifecycle management was fragmented. Onboarding relied on manual HR/IT coordination. Offboarding had gaps in device retrieval and access revocation. Mid-lifecycle events — transfers, manager changes, title updates — weren’t tracked at all.

The Approach

  • Rebuilt onboarding Slack bot with real-time status and hiring manager self-service
  • Built automated data ingestion pipeline with HR for real-time lifecycle events
  • Expanded tracking to 7 additional events: transfers, manager/title/location changes, rehires, conversions
  • Automated termination workflows: device lock, retrieval coordination, ownership handoff

Outcomes

  • ✔ 40% reduction in Day 1 onboarding delays
  • ✔ 7 additional lifecycle events automated
  • ✔ Self-service capabilities for hiring managers
  • ✔ Eliminated manual HR/IT coordination for routine events
